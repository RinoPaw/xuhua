# 叙华（Xuhua）AI 技术路线

### 1. 基于资料的 RAG 问答

xuhua 目前围绕 805 条非遗资料构建知识库，目标不是让模型凭空回答，而是先从本地资料中找到相关内容，再生成回答。检索部分结合关键词匹配和 embedding 语义检索，让用户即使不用完全准确的名称，也能找到相关非遗项目。

### 2. 从"问答系统"升级为任务型 Agent

项目正在从单一问答，逐步改造成能处理不同任务的非遗辅助工具。用户输入后，系统会先判断需求类型，比如查资料、筛选项目、做推荐、比较差异、生成展览或研学内容，再进入对应的处理流程。

### 3. 查询分析与结构化信息利用

非遗资料中包含省份、城市、级别、展示形式、主要特色、重要价值等信息。项目会把这些隐藏在文本里的内容提取成结构化字段，用于筛选、推荐和排序，而不是只依赖简单关键词搜索。

### 4. 证据型回答与内容生成

对于开放性问题，系统会把召回到的资料作为上下文，再调用模型生成回答。回答会尽量基于已有资料，并在推荐、对比、展览介绍、研学问题等场景中给出更可用的内容，避免脱离资料自由发挥。

### 5. 评估与后续迭代

后续会通过测试问题集来评估意图识别、检索命中、回答是否基于资料、推荐理由是否合理等效果。申请 MiMo Token Plan 主要是希望测试 MiMo 在长上下文代码分析、RAG 问答、中文文化资料整理和 Agent 工作流中的实际表现。
